# Security Checklist

## Инъекции
- [ ] SQL-запросы используют параметризацию (нет конкатенации строк)
- [ ] Пользовательский ввод экранирован в HTML/JSON/XML
- [ ] Команды ОС не формируются из пользовательского ввода
- [ ] LDAP/OS command/XXE инъекции исключены

## Секреты
- [ ] Нет секретов в коде (токены, пароли, ключи API)
- [ ] Секреты передаются через переменные окружения или vault
- [ ] .env и credentials-файлы в .gitignore
- [ ] Ротация секретов настроена

## Аутентификация и авторизация
- [ ] Все эндпоинты проверяют аутентификацию (кроме явно публичных)
- [ ] Авторизация проверяется на каждый запрос
- [ ] Роли/разрешения проверяются на сервере (не только на клиенте)
- [ ] Токены имеют разумный TTL и механизм отзыва

## Уязвимые зависимости
- [ ] `pip audit` / `safety check` не показывает known vulnerabilities
- [ ] Зависимости закреплены (lock-файл)
- [ ] Регулярное обновление зависимостей настроено
- [ ] Критические CVE отслеживаются

## Данные
- [ ] PII не логируется
- [ ] Чувствительные данные шифруются at rest и in transit
- [ ] Валидация ввода на всех границах системы
- [ ] Нет утечки данных в сообщениях об ошибках
