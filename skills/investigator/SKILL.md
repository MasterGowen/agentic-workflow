# Investigator

## Description
Режим диагностики. Включается при: конкретная ошибка, медленность, работа с секретами, «не видно что происходит».

## Inputs
- Описание проблемы (ошибка, симптом, метрика)
- Логи, трейсы, дампы
- Контекст: когда началось, что изменилось

## Outputs
- Найденная корневая причина
- Проверенная гипотеза
- Рекомендация по исправлению или готовый фикс

## Workflow
1. **Detect** — зафиксировать симптом, собрать контекст
2. **Analyse** — воспроизвести проблему, изолировать область
3. **Decide** — сформулировать гипотезу о причине
4. **Execute** — провести бисекцию, проверить гипотезу
5. **Verify** — подтвердить, что причина найдена, исключить альтернативы
6. **Finish** — выдать рекомендацию или передать фикс в developer/

## Escalation
- Найден баг, нужен фикс → `developer/`
- Проблема производительности → `developer/` + `../../_shared/references/postgres/queries.md`
- Нужно перепроектировать компонент → `architect/`
- Нужно проверить безопасность → `checklists/security-checklist.md`

## Exit criteria
- [ ] Корневая причина найдена
- [ ] Гипотеза проверена (воспроизводится/опровергнута)
- [ ] Рекомендация или фикс выданы
- [ ] Исключены альтернативные причины

## Non-goals
- НЕ писать новую фичу
- НЕ делать превентивную проверку без известного дефекта
- НЕ проводить code review
- НЕ проектировать архитектуру
