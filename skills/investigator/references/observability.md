# Observability — Наблюдаемость системы

## Логирование
- Используй структурированные логи (JSON)
- Уровни: DEBUG (детали), INFO (события), WARNING (аномалии), ERROR (сбои)
- Включай correlation_id для отслеживания цепочки вызовов
- Не логируй секреты и PII

## Метрики
- **RED** (для сервисов): Rate, Errors, Duration
- **USE** (для ресурсов): Utilization, Saturation, Errors
- Ключевые бизнес-метрики
- Настрой алерты на аномалии метрик

## Трейсинг
- Используй распределённый трейсинг (OpenTelemetry)
- Трейсируй все внешние вызовы (БД, HTTP, очереди)
- Добавляй spans для ключевых бизнес-операций
- Связывай трейсы с логами через trace_id

## Health Checks
- `/health/live` — жив ли процесс (liveness)
- `/health/ready` — готов ли принимать запросы (readiness)
- Проверяй зависимости: БД, кэш, внешние сервисы
- Не делай health check слишком тяжёлым
