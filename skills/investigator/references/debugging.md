# Debugging — Методология диагностики

## Воспроизведение
- Найди минимальные шаги для воспроизведения
- Зафиксируй окружение (OS, версии, конфигурация)
- Если не воспроизводится локально — собери максимум данных с проблемного окружения

## Изоляция
- Определи границы проблемы: какой компонент/модуль/функция
- Отключи нерелевантные части системы
- Используй минимальный repro-кейс

## Бисекция
- Раздели область поиска пополам
- Проверь, в какой половине проблема
- Повторяй до изоляции конкретной строки/условия
- Для регрессий: git bisect

## Анализ логов
- Ищи первое появление ошибки (не каскадные)
- Сравни логи успешного и неуспешного сценариев
- Обрати внимание на временные метки и таймауты
- Проверь correlation id для распределённых систем

## Проверка гипотезы
- Сформулируй гипотезу: «если X, то Y»
- Измени одну переменную и проверь
- Если гипотеза не подтвердилась — задокументируй и перейди к следующей
- Не проверяй несколько гипотез одновременно
