# Legacy Code — Работа с устаревшим кодом

## Что можно встретить
- Устаревшие паттерны (например, синхронные вызовы в async-контексте)
- Deprecated API (старые версии библиотек, удалённые эндпоинты)
- Отсутствие типизации
- Глобальное состояние и синглтоны
- Бизнес-логика в обработчиках (fat controllers)
- Прямые SQL-запросы вместо ORM

## Как работать, не ломая
1. **Пойми поведение** — напиши characterization-тесты перед изменениями
2. **Изолируй изменение** — минимальный diff, не рефактори попутно
3. **Следуй стилю файла** — даже если он не соответствует общим конвенциям
4. **Не смешивай** — новый код в новом стиле, старый не трогай без необходимости
5. **Документируй расхождения** — если старый код делает неочевидные вещи

## Когда рефакторить
- Только когда есть покрытие тестами
- Только когда изменение необходимо для текущей задачи
- Только с явного согласия (ADR или задача на рефакторинг)

## Антипаттерны
- «Заодно поправлю» — не делай этого
- «Перепишу с нуля» — почти никогда не оправдано
- «Добавлю ещё один if» — лучше выделить стратегию
