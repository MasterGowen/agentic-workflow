# Code Quality Checklist

## Стиль
- [ ] Код соответствует стилю проекта (ruff/black)
- [ ] Имена переменных, функций, классов — осмысленные и самодокументированные
- [ ] Длина функций ≤ 30 строк (или обоснованное исключение)
- [ ] Нет закомментированного кода

## Типизация
- [ ] Все публичные функции имеют аннотации типов
- [ ] Нет `Any` без явной необходимости
- [ ] Используются корректные generic-типы
- [ ] mypy/pyright проходит без ошибок

## Обработка ошибок
- [ ] Все внешние вызовы (БД, HTTP, файлы) обёрнуты в try/except
- [ ] Исключения специфичны (не `except Exception`)
- [ ] Ошибки логируются с достаточным контекстом
- [ ] Пользовательские ошибки наследуются от базового класса проекта

## Тесты
- [ ] Новая функциональность покрыта тестами
- [ ] Тесты проходят
- [ ] Нет пропущенных тестов (`skip` без причины)
- [ ] Граничные случаи покрыты

## Документация
- [ ] Публичные функции/классы имеют docstrings
- [ ] Docstrings описывают: что делает, параметры, возврат, исключения
- [ ] README обновлён при изменении публичного API

## Безопасность
- [ ] Нет секретов в коде
- [ ] Пользовательский ввод валидирован
- [ ] SQL-запросы используют параметризацию
- [ ] Нет уязвимых зависимостей
