# Memory Keeper

## Description
Режим работы с памятью. Включается при: обнаружен факт долгого действия, старт работы над известным проектом, завершение сессии.

## Inputs
- Тип данных: постоянный факт или состояние задачи
- Контекст текущей сессии
- История предыдущих взаимодействий с проектом

## Outputs
- Факт, записанный в правильном формате
- Состояние задачи зафиксировано
- Resume-point для следующей сессии

## Workflow
1. **Detect** — определить, что достойно сохранения (факт vs состояние)
2. **Analyse** — проверить, нет ли уже такого факта, не устарел ли
3. **Decide** — выбрать формат записи согласно memory-policy
4. **Execute** — записать факт или состояние задачи
5. **Verify** — проверить корректность формата и отсутствие дубликатов
6. **Finish** — подтвердить сохранение

## Escalation
- Нужен MCP-сервер для хранения → `developer/` + `../../references/mcp/server-design.md`
- Нужно спроектировать схему хранения → `architect/`

## Exit criteria
- [ ] Факт записан в правильном формате
- [ ] Состояние задачи зафиксировано (если применимо)
- [ ] Resume-point определён для следующей сессии
- [ ] Проверено отсутствие дубликатов

## Non-goals
- НЕ хранить данные (это делает MCP-механизм)
- НЕ писать MCP-серверы
- НЕ проектировать схему БД для памяти
- НЕ анализировать/агрегировать сохранённые факты
