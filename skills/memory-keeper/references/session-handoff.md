# Session Handoff — Фиксация состояния задачи

## Когда фиксировать
- Завершение сессии (явное или по таймауту)
- Достижение промежуточного milestone
- Переключение на другую задачу
- Обнаружение блокирующего фактора

## Что фиксировать
- **Идентификатор задачи** — номер issue/тикета или краткое описание
- **Текущий шаг** — на каком этапе workflow остановились
- **Что сделано** — конкретные результаты (файлы, коммиты, решения)
- **Что осталось** — следующие шаги с приоритетами
- **Блокирующие факторы** — что мешает продолжить
- **Resume-point** — с чего начать следующую сессию (конкретный файл/функция/действие)

## Формат
```markdown
## Session Handoff: [Задача]

**Status**: IN_PROGRESS | BLOCKED | READY_FOR_REVIEW
**Last update**: [ISO 8601]

### Completed
- [x] Конкретный результат 1
- [x] Конкретный результат 2

### Remaining
- [ ] Следующий шаг 1 (priority: HIGH)
- [ ] Следующий шаг 2 (priority: MEDIUM)

### Blockers
- Блокер 1: описание и что нужно для разблокировки

### Resume Point
Начни с файла `path/to/file.py`, функция `function_name()`.
```

## Антипаттерны
- «Продолжить работу» — недостаточно конкретно
- Слишком длинный список remaining (>7 пунктов) — декомпозируй
- Отсутствие resume-point — следующий агент не поймёт, с чего начать
