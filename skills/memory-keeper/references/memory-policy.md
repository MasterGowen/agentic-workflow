# Memory Policy — Что, когда и как сохранять

## Что достойно сохранения
- **Постоянные факты**: архитектурные решения, конвенции проекта, предпочтения пользователя, известные ограничения
- **Состояние задачи**: текущий шаг, что сделано, что осталось, блокирующие факторы
- **Контекст**: используемые технологии, версии, конфигурация окружения

## Что НЕ сохранять
- Промежуточные результаты (временные файлы, логи сборки)
- Очевидные вещи (язык проекта, стандартные конвенции)
- Устаревшую информацию (без пометки об устаревании)

## Формат записи
```
[Тип: FACT | TASK_STATE | CONTEXT]
[Теги: через, запятую]
[Timestamp: ISO 8601]
Содержание: ...
```

## Когда читать
- При старте работы над известным проектом — загрузить все факты
- При смене контекста — загрузить состояние незавершённых задач

## Когда обновлять
- При изменении факта (новая версия библиотеки, смена конвенции)
- При прогрессе задачи (новый статус, выполненный шаг)

## Когда инвалидировать
- Факт больше не актуален (пометить `[DEPRECATED]`, не удалять)
- Задача завершена (пометить `[DONE]`)
- Информация противоречит новым данным (заменить, старую пометить)
