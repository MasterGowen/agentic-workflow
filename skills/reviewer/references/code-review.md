# Code Review — Оценка diff/PR

## Корректность
- Код делает то, что заявлено в описании PR
- Все граничные случаи обработаны
- Нет off-by-one ошибок, null pointer, race conditions
- Бизнес-логика соответствует требованиям

## Конвенции
- Код следует стилю проекта
- Именование соответствует принятым соглашениям
- Структура файлов/пакетов соответствует архитектуре
- Используются принятые в проекте паттерны

## Читаемость
- Код понятен без дополнительных пояснений
- Сложные участки имеют комментарии (не что, а почему)
- Нет избыточной вложенности
- Функции делают ровно одну вещь

## Риски
- Изменения не ломают обратную совместимость (или это явно задокументировано)
- Миграции БД обратимы
- Новые зависимости обоснованы
- Нет скрытых сайд-эффектов

## Процесс ревью
1. Прочитай описание PR — пойми, что должно быть сделано
2. Посмотри общую картину diff — оцени масштаб изменений
3. Пройди по файлам — от критичных к второстепенным
4. Оставь замечания — конкретные, actionable, с приоритетом
5. Вынеси вердикт: approve / request changes / comment
