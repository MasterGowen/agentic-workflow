# Refactoring — Безопасное изменение структуры

## Принципы
- Изменяй структуру, не изменяя поведение
- Каждый рефакторинг — отдельный коммит
- Тесты должны проходить до и после каждого шага
- Не смешивай рефакторинг и новую функциональность

## Типовые рефакторинги
- **Extract Method** — выделение повторяющегося кода в функцию
- **Extract Class** — разделение класса с множеством ответственностей
- **Replace Conditional with Polymorphism** — замена цепочек if/else на стратегии
- **Introduce Parameter Object** — группировка связанных параметров
- **Replace Magic Number with Constant** — замена литералов на именованные константы

## Когда НЕ рефакторить
- Нет тестового покрытия
- Код на продакшене без возможности быстрого отката
- Сроки не позволяют провести полное тестирование
- Изменение не несёт измеримой пользы

## Оценка рефакторинга
- [ ] Поведение не изменилось (тесты проходят)
- [ ] Код стал понятнее
- [ ] Зависимости стали явными
- [ ] Новые абстракции оправданы
- [ ] Нет преждевременной оптимизации
