# PR Review Checklist

## Описание PR
- [ ] PR имеет осмысленное название
- [ ] Описание объясняет: что, зачем, как
- [ ] Связанные задачи/тикеты указаны

## Корректность
- [ ] Код решает заявленную проблему
- [ ] Граничные случаи обработаны
- [ ] Обработка ошибок присутствует
- [ ] Нет гонок данных (race conditions)

## Стиль и конвенции
- [ ] Код соответствует стилю проекта
- [ ] Именование понятное и последовательное
- [ ] Нет закомментированного кода
- [ ] Нет отладочных print/console.log

## Тесты
- [ ] Новый код покрыт тестами
- [ ] Существующие тесты проходят
- [ ] Тесты проверяют поведение, а не реализацию
- [ ] Нет хрупких тестов (завязанных на порядок/время)

## Безопасность
- [ ] Нет секретов в коде
- [ ] Пользовательский ввод валидирован
- [ ] SQL-запросы параметризованы
- [ ] Права доступа проверены

## Производительность
- [ ] Нет N+1 запросов
- [ ] Нет блокирующих операций в async-контексте
- [ ] Запросы к БД оптимальны (индексы, объёмы)

## Документация
- [ ] Docstrings для публичного API
- [ ] README обновлён при необходимости
- [ ] CHANGELOG обновлён при необходимости
