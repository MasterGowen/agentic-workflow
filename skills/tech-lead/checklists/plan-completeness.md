# Plan Completeness Checklist

Проверка плана перед передачей в developer/. Прогоняется на шаге Finish.

## Покрытие
- [ ] Все компоненты из ADR разбиты на задачи
- [ ] Нет задач без границ (что входит / что не входит)
- [ ] Выбран паттерн workflow (или комбинация ≤2)
- [ ] Паттерн обоснован классификацией (Cynefin × связанность)

## Зависимости и параллелизм
- [ ] Зависимости указаны явно (сначала X, потом Y)
- [ ] Параллельные группы выделены
- [ ] WIP-лимит обозначен (не более 3–4 параллельных на точку интеграции)
- [ ] Критический путь / критическая цепь определены

## Интеграция
- [ ] Точки интеграции запланированы
- [ ] В каждой точке — контракт-проверка (реальные интерфейсы vs ADR)
- [ ] Частота интеграции соответствует риску рассинхрона

## Риски
- [ ] Риски из ADR наложены на критическую цепь
- [ ] Для High-рисков на цепи — буфер или fallback
- [ ] Необратимые точки выделены отдельной задачей-проверкой

## Встроенные этапы
- [ ] Ревью включено в план
- [ ] Тесты включены в план
- [ ] Документация включена в план

## Границы
- [ ] План не декомпозирует до уровня функций (это developer)
- [ ] План не назначает конкретных исполнителей
- [ ] План не оценивает трудозатраты в часах
