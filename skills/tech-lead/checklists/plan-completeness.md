# Plan Completeness Checklist

Проверка перед передачей в developer/. Прогоняется на шаге Finish.

## Спецификация (tech spec)
- [ ] Есть ERD для всех новых хранилищ (Qdrant payload, SQLite)
- [ ] Есть class-диаграммы для всех новых модулей с сигнатурами методов
- [ ] Есть sequence-диаграммы для всех pipeline'ов (write, search, recall)
- [ ] Есть таблица MCP API — все инструменты с сигнатурами
- [ ] Есть план миграции с таблицей значений для backfill
- [ ] Есть конфигурация — все новые ENV-переменные
- [ ] Есть файловая структура — новые и изменённые файлы
- [ ] Есть принципы, которые developer не должен нарушать
- [ ] В спецификации нет кода — только формальные модели и таблицы

## Бэклог (backlog)
- [ ] Все компоненты из ADR разбиты на задачи
- [ ] Каждая задача имеет: что сделать, ⛓️ зависимости, 📄 файлы, ✅ критерии, 🚫 границы, 🧪 тесты, 📝 документация
- [ ] Зависимости указаны явно (сначала X, потом Y)
- [ ] Параллельные группы выделены
- [ ] WIP-лимит обозначен (не более 3–4 параллельных на точку интеграции)
- [ ] Критический путь определён
- [ ] Есть граф зависимостей (ASCII)

## Чего НЕТ в бэклоге (проверка на засорение)
- [ ] Нет рисков
- [ ] Нет чек-листов
- [ ] Нет архитектурных рассуждений
- [ ] Нет точек интеграции
- [ ] Нет cognitive review
- [ ] Нет «замечаний для developer» в конце
- [ ] Нет оценки трудозатрат в часах
- [ ] Нет конкретных исполнителей

## Встроенные этапы
- [ ] Ревью включено (отдельная задача или фаза)
- [ ] Тесты включены (отдельная задача или в каждой задаче)
- [ ] Документация включена (в каждой задаче, где нужно)

## Границы
- [ ] План не декомпозирует до уровня функций (это developer)
- [ ] Спецификация и бэклог — два разных файла, не смешаны
