# Infra

## Description
Режим инфраструктуры. Включается при: настройка/правка CI, релиз версии, откат, staging/prod окружения.

## Inputs
- Тип задачи: CI, деплой, Docker для прода, релиз, откат
- Конфигурация окружения
- Кодовая база (для CI/сборки)

## Outputs
- Настроенный пайплайн / выпущенный релиз / выполненный откат
- Проверка в staging

## Workflow
1. **Detect** — определить тип инфраструктурной задачи
2. **Analyse** — изучить текущую конфигурацию, окружения, зависимости
3. **Decide** — выбрать соответствующий reference
4. **Execute** — применить изменения
5. **Verify** — проверить в staging-окружении
6. **Finish** — зафиксировать результат, обновить документацию

## Escalation
- Нужна локальная разработка → `developer/`
- Нужна диагностика проблемы → `investigator/`
- Нужно спроектировать инфраструктуру → `architect/`

## Exit criteria
- [ ] Пайплайн настроен / релиз выпущен / откат выполнен
- [ ] Проверено в staging-окружении
- [ ] Чек-лист релиза пройден (если применимо)
- [ ] Документация инфраструктуры обновлена

## Non-goals
- НЕ локальная разработка
- НЕ написание кода приложения
- НЕ проектирование архитектуры приложения
- НЕ диагностика багов приложения
