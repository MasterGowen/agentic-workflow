# CI/CD — Пайплайны сборки, тестов, линта, деплоя

## Структура пайплайна
1. **Lint & Format** — быстро, блокирует при ошибках
2. **Type Check** — mypy/pyright
3. **Unit Tests** — параллельно, быстро
4. **Integration Tests** — с тестовой БД и сервисами
5. **Build** — сборка Docker-образа
6. **Deploy to Staging** — автоматически при merge в main
7. **Deploy to Production** — manual trigger или по тегу

## Принципы
- Fail fast: линтер и типизация до тестов
- Кэширование зависимостей между запусками
- Параллельные шаги, где нет зависимостей
- Детерминированные сборки (закреплённые версии)
- Артефакты сборки неизменяемы

## Инструменты
- GitHub Actions / GitLab CI / Jenkins
- ruff — линтер и форматтер
- mypy — проверка типов
- pytest — тесты
- Docker BuildKit — сборка образов

## Антипаттерны
- Секреты в логах CI
- Долгие пайплайны (>15 мин)
- Пропуск тестов «потому что долго»
- Разные команды сборки локально и в CI
