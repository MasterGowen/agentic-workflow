# Deployment — Стратегия релиза, окружения, rollback, секреты

## Окружения
- **Development** — локальная разработка, hot reload
- **Staging** — максимально похоже на prod, для проверки перед релизом
- **Production** — боевое окружение, минимальные привилегии

## Стратегия релиза
- **Blue-Green**: два окружения, переключение трафика
- **Rolling**: постепенная замена инстансов
- **Canary**: малая доля трафика на новой версии
- Выбор зависит от допустимого времени простоя и сложности отката

## Rollback
- Всегда иметь план отката до релиза
- Миграции БД должны быть обратимы (down-миграция)
- Хранить последние N успешных артефактов
- Автоматический rollback при превышении порога ошибок

## Секреты
- Никогда не хранить в коде или образах
- Использовать vault/секреты CI/облачный secrets manager
- Ротировать регулярно
- Разные секреты для разных окружений

## Health Checks
- Liveness: процесс жив
- Readiness: готов принимать трафик (БД доступна, миграции применены)
- Startup: начальная инициализация завершена
