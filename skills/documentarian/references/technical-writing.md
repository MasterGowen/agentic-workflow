# Technical Writing — README, docstrings, руководства

## README
- **Что**: название и однострочное описание
- **Зачем**: какую проблему решает
- **Как**: quick start (установка, минимальный пример)
- **Где**: ссылки на полную документацию
- **Кто**: контрибьюторы, лицензия

## Docstrings
- **Функции**: что делает, параметры, возврат, исключения
- **Классы**: назначение, атрибуты, основные методы
- **Модули**: общее описание и ключевые экспорты
- Формат: Google-style или Sphinx (единообразно в проекте)

## Руководства
- **Tutorial**: пошаговое обучение с нуля
- **How-to**: решение конкретной задачи
- **Reference**: полное описание API
- **Explanation**: почему так устроено, архитектурные решения

## Принципы
- Пиши для целевой аудитории (новичок / опытный / админ)
- Один документ — одна цель
- Примеры кода должны быть рабочими
- Избегай «очевидно», «просто», «легко»
- Обновляй документацию вместе с кодом
