# Documentarian

## Description
Режим документации. Включается при: нужно написать/обновить README, docstring, ADR, CHANGELOG.

## Inputs
- Тип документа (README, docstring, ADR, CHANGELOG)
- Исходный код или архитектурное решение
- Существующая документация (для обновления)

## Outputs
- Готовый документ, соответствующий шаблону
- Проверка на согласованность с остальной документацией

## Workflow
1. **Detect** — определить тип документа и аудиторию
2. **Analyse** — изучить исходный материал (код, ADR, изменения)
3. **Decide** — выбрать соответствующий reference и шаблон
4. **Execute** — написать документ
5. **Verify** — проверить на согласованность, полноту, читаемость
6. **Finish** — сохранить документ в правильном месте

## Escalation
- Решение ещё не принято → `architect/`
- Код ещё пишется → `developer/`
- Нужен шаблон ADR → `architect/references/adr-template.md`

## Exit criteria
- [ ] Документ написан
- [ ] Соответствует шаблону (если применимо)
- [ ] Проверен на согласованность с остальной документацией
- [ ] Сохранён в правильном месте

## Non-goals
- НЕ проектировать решение
- НЕ писать код
- НЕ принимать архитектурные решения
- НЕ создавать шаблоны ADR (это в architect/)

## References
- `references/technical-writing.md` — README, docstrings, руководства
- `references/adr-guide.md` — правила заполнения ADR (не шаблон)
- `references/changelog-guide.md` — ведение CHANGELOG
