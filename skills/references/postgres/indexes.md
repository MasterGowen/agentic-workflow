# PostgreSQL — Indexes

## Типы индексов
- **B-tree** (по умолчанию): `=`, `<`, `>`, `BETWEEN`, `ORDER BY`
- **Hash**: только `=`, быстрее B-tree для больших ключей
- **GIN**: `@>`, `?`, `@@` для JSONB, массивов, full-text search
- **GiST**: геоданные, full-text search
- **BRIN**: большие таблицы с коррелированными данными (временные ряды)

## Когда создавать индекс
- Колонка используется в `WHERE`, `JOIN`, `ORDER BY`
- Запрос выполняется часто и медленно (подтверждено EXPLAIN)
- Кардинальность колонки высокая (>10% уникальных значений)
- Не создавать на каждую FK-колонку автоматически — только если есть медленные JOIN

## Когда НЕ создавать
- Маленькие таблицы (<1000 строк)
- Колонки с низкой кардинальностью (пол, статус из 3 значений)
- Редко используемые запросы
- Таблицы с интенсивной записью (каждый индекс замедляет INSERT/UPDATE)

## Частичные индексы
```sql
-- Индекс только для активных записей
CREATE INDEX idx_active_orders ON orders (user_id) WHERE status = 'active';
```

## Составные индексы
- Порядок колонок: от самой селективной к менее селективной
- Индекс `(a, b)` покрывает запросы по `a` и по `a, b`, но НЕ по `b`

## Мониторинг
- `pg_stat_user_indexes` — использование индексов
- `pg_stat_user_tables` — seq scan vs index scan
- Неиспользуемые индексы — кандидаты на удаление
