# MCP — Server Design

## Архитектура MCP-сервера
```
MCP Server
├── tools/       # Инструменты, доступные агенту
├── resources/   # Данные, доступные для чтения
├── prompts/     # Шаблоны промптов
└── transport/   # stdio / SSE / Streamable HTTP
```

## Схема Tool
```python
{
    "name": "search_docs",
    "description": "Search documentation by keyword",
    "inputSchema": {
        "type": "object",
        "properties": {
            "query": {"type": "string", "description": "Search query"},
            "limit": {"type": "integer", "default": 10}
        },
        "required": ["query"]
    }
}
```

## Транспорт
- **stdio**: процесс запускается агентом, общение через stdin/stdout
- **SSE**: сервер запущен отдельно, общение через Server-Sent Events
- **Streamable HTTP**: новый стандарт, замена SSE

## Принципы проектирования
- Один инструмент — одна чёткая функция
- Описание инструмента должно быть достаточным для агента
- Валидация входных параметров на стороне сервера
- Идемпотентность для мутирующих операций
- Таймауты на все операции

## Безопасность
- Валидация всех входных данных
- Ограничение доступа к файловой системе
- Нет выполнения произвольного кода
- Аутентификация для SSE/HTTP транспорта

## Антипаттерны
- Слишком много инструментов (>20) — агент теряется
- Инструменты с пересекающейся функциональностью
- Тяжёлые операции без индикации прогресса
- Отсутствие обработки ошибок в инструментах
