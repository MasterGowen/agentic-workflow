# Architect

## Description
Режим проектирования. Включается при: новый модуль/сервис/API, выбор между ≥2 вариантами, непонятный scope.

## Inputs
- Описание задачи или требования
- Кодовая база (для анализа)
- Контекст существующей архитектуры

## Outputs
- Выбранный архитектурный вариант
- Заполненный ADR
- Список рисков
- Декомпозированный план

## Workflow
1. **Detect** — определить scope задачи, затронутые компоненты
2. **Analyse** — проанализировать существующий код, зависимости, контракты
3. **Decide** — сгенерировать ≥2 варианта, применить tradeoffs, выбрать решение
4. **Execute** — заполнить ADR, зафиксировать решение
5. **Verify** — проверить согласованность по чек-листу архитектуры
6. **Finish** — передать план в developer/

## Escalation
- Нужна диагностика конкретной проблемы → `investigator/`
- Задача тривиальна и не требует проектирования → `developer/`
- Нужно задокументировать решение → `documentarian/`

## Exit criteria
- [ ] Выбран один архитектурный вариант
- [ ] ADR заполнен (статус, контекст, решение, альтернативы, последствия)
- [ ] Риски перечислены и оценены
- [ ] План декомпозирован на конкретные шаги
- [ ] Чек-лист архитектуры пройден

## Non-goals
- НЕ писать код
- НЕ исправлять баги
- НЕ делать code review
- НЕ писать документацию пользователя

## Decision heuristics
Совместимость > Простота > Минимальный diff > Производительность
