# Architecture Review Checklist

## Компоненты
- [ ] Все компоненты системы описаны
- [ ] Для каждого компонента указаны: ответственность, интерфейсы, зависимости
- [ ] Нет неописанных компонентов

## Зависимости
- [ ] Все зависимости явно перечислены
- [ ] Нет циклических зависимостей
- [ ] Направление зависимостей соответствует слоям архитектуры
- [ ] Нет скрытых/неявных зависимостей

## Контракты
- [ ] Все API-контракты согласованы (форматы, протоколы, версионирование)
- [ ] Контракты БД согласованы (схема, миграции, индексы)
- [ ] События/сообщения имеют определённую схему

## Нефункциональные требования
- [ ] Производительность: оценены узкие места
- [ ] Безопасность: определена модель угроз
- [ ] Масштабируемость: описаны пределы текущего решения
- [ ] Отказоустойчивость: описана стратегия обработки сбоев

## Согласованность
- [ ] Решение не противоречит существующим ADR
- [ ] Решение соответствует общим принципам архитектуры проекта
- [ ] Все заинтересованные стороны уведомлены
