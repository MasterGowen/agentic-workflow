# ADR Template — Шаблон архитектурного решения

## Структура ADR

### Статус
- `proposed` — предложено, ожидает утверждения
- `accepted` — принято, реализуется
- `deprecated` — заменено новым решением
- `superseded` — заменено, см. ADR-XXX

### Контекст
- Опиши текущую ситуацию и проблему
- Почему нужно принимать решение именно сейчас
- Какие ограничения действуют (технические, временные, организационные)

### Решение
- Опиши выбранный подход
- Приведи ключевые детали реализации
- Укажи, что именно меняется

### Альтернативы
- Перечисли рассмотренные альтернативы
- Для каждой: краткое описание и причина отклонения
- Минимум 2 альтернативы

### Последствия
- **Положительные**: что улучшится
- **Отрицательные**: чем придётся пожертвовать
- **Нейтральные**: что изменится, но без явного +/- эффекта

## Правила
- Один ADR = одно решение
- Нумерация: ADR-001, ADR-002, ...
- Не удалять старые ADR — менять статус
- Ссылаться на связанные ADR
