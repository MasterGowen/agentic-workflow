# Planning — Декомпозиция задачи

## Scope
- Определи границы задачи: что входит, что нет
- Зафиксируй явные и неявные требования
- Выяви затронутые компоненты и их владельцев

## Критерии готовности
- Сформулируй измеримые критерии завершения
- Для каждого критерия укажи способ проверки
- Критерии должны быть бинарными (да/нет)

## Риски
- Перечисли известные риски (технические, организационные, временные)
- Оцени каждый риск: вероятность × влияние
- Для High-рисков предложи mitigation

## Разбивка на шаги
- Декомпозируй задачу на независимые шаги
- Каждый шаг должен иметь чёткий вход и выход
- Укажи зависимости между шагами
- Оцени сложность каждого шага (S/M/L)

## Что НЕ входит
- Выбор технического решения → `tradeoffs.md`
- Фиксация решения → `adr-template.md`
- Написание кода → `developer/`
